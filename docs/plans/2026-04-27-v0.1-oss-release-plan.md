## Path to 9+ rating

Project currently rates **~8.5/10** post quality pass. Below is the prioritized work to move the needle further.

### P1 — Quality drift ✅ Complete

1. [x] **Fix Toolbar width + animations.** Content-sized, no animation, uniform item padding. _(#42 + follow-up commits, 2026-04-28)_
2. [x] **Document accent color usage.** `DESIGN.md` written; accent rule + selection state rule enforced across all 36 components. _(#42, 2026-04-28)_
3. [x] **Scrollbar consistency.** Sidebar routed through `ScrollArea`; Base UI primitive viewports documented as native exceptions per `DESIGN.md`. _(#42, 2026-04-28)_
4. [x] **Accessibility audit + fixes.** Full static audit (light + dark mode), easy wins applied, findings doc at `docs/plans/2026-04-28-a11y-audit-findings.md`. _(#42, 2026-04-28)_

### P2 — Parity + scaffolding

5. [x] **Context Menu component.** Full Base UI compound API shipped. 35 → 36 components. _(#42, 2026-04-28)_
6. [ ] **CLI expansion.** Currently scaffolds only `button` + `accordion`. Expand `add` to support all 36 components. Needs architectural pass — bundle component sources into CLI tarball at build time. _Separate sprint._

### P3 — Test depth ✅ Complete

7. [x] **Real RTL component tests.** jsdom + `@testing-library/react` + `@testing-library/user-event` set up. Render + ARIA assertions on all 36 components. _(#46, 2026-04-28)_
8. [x] **axe-core a11y assertions per component.** `jest-axe` wired; `await axe(container)` on every component tree. _(#46, 2026-04-28)_

### P3 — Additional quality ✅ Complete

- [x] **Contrast token pass (dark + light).** 5 new tokens; border, focus ring, track, destructive-text tuned for AA. _(#43, 2026-04-28)_
- [x] **Opacity-disabled migration.** 14 components migrated from `opacity: 0.64` to muted tokens. _(#45, 2026-04-28)_
- [x] **Interaction depth backfill.** 18 components with real user-event tests; 236 total tests. _(#48, 2026-04-28)_

### P4 — Confidence + differentiation (longer arc)

9. [ ] **Visual regression baseline (Chromatic / Percy).** CI snapshot every component × variant × state. Block PRs on unintended diffs. Locks in design quality.
10. [ ] **Theme studio.** Interactive token editor in the docs site → exports a custom theme file. Genuinely differentiating feature, plays to the StyleX strength, marketing-friendly.

### P5 — Doesn't engineer, has to happen

11. [ ] **Adoption signal.** Real users, GitHub stars, npm downloads, external issues. Drives the rating from "good library no one uses" to "good library people use." Comes from launch + community + content, not code.

## Beyond v0.1.0 — Milestone Backlog

To exceed this release and earn a v0.2 / v1.0:

### v0.2 — Completeness

- [x] **Context Menu component.** Shipped. _(#42, 2026-04-28)_
- [ ] **Docs bundle size <500KB gzipped.** Currently 1.3MB raw / 372KB gzipped, dominated by Shiki bundling every grammar. Lazy-load grammars and themes on demand, or swap to Starry Night.
- [x] **Disabled-state token consistency.** All 14 components migrated from opacity to muted tokens. _(#45, 2026-04-28)_
- [x] **Accessibility test suite.** RTL + axe-core on all 36 components. _(#46 + #48, 2026-04-28)_

### v0.3 — Confidence

- [ ] **Visual regression baseline.** Chromatic or Percy in CI. Snapshot every component in every variant/state. Block PRs on unintended visual diffs.
- [ ] **Example integrations.** `examples/nextjs/`, `examples/remix/`, `examples/vite-react/`. Each is a deployable minimal app demonstrating install → use → theme.
- [ ] **Animation guide.** Doc page connecting `@basex-ui/intelligence` animation presets to component usage. "Use this preset for that interaction" with live demos.

### v1.0 — Polish & adoption

- [ ] **Type-level docs.** TSDoc on every public prop so hover hints in editors are useful.
- [ ] **Migration guides.** From shadcn/ui, from Radix, from Headless UI.
- [ ] **Theme studio.** Interactive token editor in the docs site that exports a custom theme file.
- [ ] **AI agent quickstart.** End-to-end walkthrough of using the MCP server from Claude / Cursor to scaffold a UI from intent.
- [ ] **Performance budget.** Per-component bundle size tracked in CI, regression alerts.
