---
title: Setup & Token Tools
description: Tools for pre-generation requirements, design tokens, themes, and animations.
---

Four tools provide the context agents need to generate correct, styled code.

## get_component_setup

Call this before generating code for any component. Returns the import statement, CSS requirements, required props, and animation presets used.

```
→ get_component_setup({ name: "Accordion" })

← {
    "component": "Accordion",
    "import": "import { Accordion } from '@basex-ui/components';",
    "cssRequirements": "@keyframes basex-accordion-panel { from { height: 0; } to { height: var(--accordion-panel-height); } }",
    "requiredProps": [
      { "part": "Item", "prop": "value", "type": "string", "note": "Unique identifier for each item" }
    ],
    "animationPresets": [
      { "interaction": "panel expand/collapse", "preset": "Expand" },
      { "interaction": "chevron rotation", "preset": "Move" }
    ]
  }
```

This prevents three common agent mistakes:
1. Missing CSS animations that components depend on
2. Omitting required props (e.g., `value` on Accordion.Item)
3. Using wrong animation timing

## get_tokens

Returns design tokens organized by category. Use these values instead of hardcoded numbers.

```
→ get_tokens({ category: "spacing" })

← {
    "category": "spacing",
    "tokens": {
      "space1": "0.25rem",
      "space2": "0.5rem",
      "space3": "0.75rem",
      "space4": "1rem",
      "space6": "1.5rem",
      "space8": "2rem",
      "space10": "2.5rem",
      "space12": "3rem"
    }
  }
```

Available categories: `colors`, `spacing`, `typography`, `radius`, `borders`, `shadows`, `motion`.

Omit the category to get all tokens at once.

## get_theme_example

Shows how to apply light, dark, or custom themes using StyleX.

```
→ get_theme_example({ theme: "custom" })

← import * as stylex from '@stylexjs/stylex';
   import { tokens } from '@basex-ui/tokens';
   import { generateSemanticColors } from '@basex-ui/tokens/oklch';

   const customColors = generateSemanticColors('#8B5CF6');

   const purpleTheme = stylex.createTheme(tokens, {
     ...customColors,
     colorFocusRing: customColors.colorPrimary,
   });

   function App({ children }) {
     return (
       <div {...stylex.props(purpleTheme)}>
         {children}
       </div>
     );
   }
```

The custom theme example uses `generateSemanticColors` from `@basex-ui/tokens/oklch` to create a full semantic palette from a single hex color. This generates all the primary, secondary, and surface colors needed by the token system.

## get_animation

Query animation presets by name or by use case description. Returns duration, easing, and the design tokens that map to those values.

```
→ get_animation({ useCase: "panel collapse" })

← {
    "matchedPreset": "Expand",
    "name": "Expand",
    "duration": "200ms",
    "durationToken": "motionDurationNormal",
    "easing": "cubic-bezier(0.4, 0, 0.2, 1)",
    "easingToken": "motionEaseInOut",
    "description": "For content revealing in place. Accelerates then decelerates.",
    "useFor": ["accordion panels", "collapsible content", "disclosure", "height transitions"]
  }
```

```
→ get_animation({ preset: "State" })

← {
    "name": "State",
    "duration": "100ms",
    "durationToken": "motionDurationFast",
    "easing": "cubic-bezier(0, 0, 0.2, 1)",
    "easingToken": "motionEaseOut",
    "description": "For instant feedback on interaction. Feels responsive.",
    "useFor": ["hover", "focus", "active", "color changes", "border changes", "background changes"]
  }
```

Omit both parameters to get all five presets at once.
