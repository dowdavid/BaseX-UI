# Security Policy

## Supported Versions

BaseX UI is pre-1.0. Security fixes will be applied to the latest published minor version only.

| Version | Supported |
| ------- | --------- |
| `0.1.x` | ✅        |
| `< 0.1` | ❌        |

## Reporting a Vulnerability

If you believe you've found a security vulnerability in any BaseX UI package, **please do not open a public issue.**

Instead, email **dow4537@gmail.com** with:

- A description of the vulnerability
- Steps to reproduce (or a minimal proof-of-concept)
- The package(s) and version(s) affected
- Any suggested mitigation, if you have one

You can expect:

- An acknowledgement within **3 business days**
- An initial assessment within **7 business days**
- A patch and coordinated disclosure plan, where applicable

We'll credit you in the release notes for the fix unless you'd prefer to remain anonymous.

## Out of Scope

- Vulnerabilities in third-party dependencies — please report those upstream first.
- Issues that require physical access to a user's device.
- Social-engineering attacks against maintainers or contributors.
