# Changelog

This is the root changelog for the BaseX UI monorepo. Per-package changelogs are auto-generated by [Changesets](https://github.com/changesets/changesets) and live alongside each package.

## 0.1.0 — 2026-04-27

Initial public release.

### `@basex-ui/components`

- 35 accessible React components built on [Base UI](https://base-ui.com) and styled with [StyleX](https://stylexjs.com): Accordion, AlertDialog, Autocomplete, Avatar, Button, Checkbox, CheckboxGroup, Collapsible, Combobox, Dialog, Drawer, Field, Fieldset, Form, Input, Menu, Menubar, Meter, NavigationMenu, NumberField, Popover, PreviewCard, Progress, Radio, ScrollArea, Select, Separator, Slider, Switch, Tabs, Toast, Toggle, ToggleGroup, Toolbar, Tooltip.
- Compound component pattern across the library (e.g. `Dialog.Root`, `Dialog.Trigger`, `Dialog.Popup`).
- `sx` prop on every component for StyleX overrides.
- Subpath imports per component (`@basex-ui/components/button`).
- Per-component machine-readable manifest (`manifest.json`) for AI discovery.

### `@basex-ui/tokens`

- OKLCH color palette, spacing scale, typography, motion, and radius tokens as StyleX variables.
- Subpath exports for `presets` and `oklch` utilities.

### `@basex-ui/styles`

- `lightTheme` and `darkTheme` StyleX themes.
- Shared `focusRing` utility used by all interactive components.

### `@basex-ui/intelligence`

- Intent resolution data: natural-language signals → component recommendations with reasoning and composition blueprints.
- Anti-pattern catalogue with suggested alternatives.
- Animation presets (State, Expand, Move, …) tied to motion tokens.

### `@basex-ui/mcp-server`

- MCP server exposing 10 tools: `list_components`, `search_components`, `get_component`, `get_component_example`, `resolve_intent`, `check_usage`, `get_tokens`, `get_theme_example`, `get_animation`, `get_component_setup`.
- Stdio transport, no API keys, no HTTP.

### `@basex-ui/cli`

- `init`, `add`, `list`, `theme` commands for scaffolding BaseX UI into a project.

### Docs site

- Vite SPA at https://basex-ui-docs.vercel.app with sidebar search, live demos, and per-component API references.
- Per-component Open Graph + Twitter Card metadata: each `/components/<slug>` URL renders a unique preview card with logomark and component name when shared.
